# Анализ проекта
## Выполненные задачи
В ходе выполнения проекта были выполнены задачи:

- Обработка DataSet по продажам автомобилей, подготовка данных к обучению модели
    - Удаление явных и скрытых дубликатов
    - Заполнение пропусков в данных
        - Пропуски числовых значений заполнены средними значениям по train выборке
        - Пропуски категориальных значений заполнены пустыми строками
    - Работа с преобразованием типов данных, чистка числовых признаков от лишних данных (размерности физических величин)
    - Удаление неинформативных столбцов

- Анализ входных данных
    - Pairplot числовых признаков
    - Матрица корреляций признаков
        - Корреляция Пирсона
        - Phik матрица
        - V-мера Крамера
    - Поиск взаимосвязей признаков по выполненным статистическим моделям
    - Построение визуализации анализа данных

- Моделирование
    - LinearRegression на числовых признаках (r2_score=0.59)
    - Стандартизация признаков
    - Регуляризация Lasso, Ridge, ElasticNet 
    - OneHotEncoding категориальных признаков
    - Подбор гиперпараметров посредством GridSearchSV c кросс-валидацией

- Разработка интерактивного интерфейса на Streamlit
    - Визуализации EDA: Pairplot, Heatmap
    - Возможность прогнозирования на данных загруженного CSV-файла на обученной модели
    - Визуализация весов

## Полученные результаты

Качество LinearRegression относительно r2_score: 0.59, что говорит об умеренной объяснительной способности модели; относительно MSE: 2,33*10^11, что говорит об огромной среднеквадратичной ошибке. Возможно, MSE показывает такой результат в виду проблемы с выбросами в данных, нелинейных зависимостях признаков. Стандартизация, Lasso-регрессия не улучшили качество модели. 

Наибольший прирост в качестве модели относительно метрики r2_score получился путём выполнения Ridge-регрессии с кросс-валидацией по десяти фолдам и подбора оптимального гиперпараметра alpha (с 0.59 до 0.65). Относительно бизнес-метрики эта модель так же лучшая.

Не удалось проработать модель с точки зрения MSE. В следующей итерации планируется Feature Engineering для улучшения качества модели.

Разработанный web-сервис прост и несёт в себе основные визуализации по каждой части проекта, а так же имеет функцию прогноза на основе загруженного CSV-файла. В следующей итерации планируется введение более детального интерфейса для взаимодействия с ручным вводом признаков и дальнейшей визуализацией прогнозов.
